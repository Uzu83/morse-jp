# morse-jp

**和文・欧文モールス符号の双方向変換ツール。** テキスト ↔ モールスを相互変換し、音声で再生、マイクからリアルタイム復号もできる Web アプリです。ブラウザだけで動き、インストール不要。

> 日本語（和文モールス／カナ）に対応しているのが特徴です。「ガ＝カ＋濁点」のような濁音・半濁音の合成、長音、ひらがな入力の自動カタカナ化まで扱います。

## 特徴

- **双方向**：テキスト → モールス、モールス → テキスト
- **和文＋欧文**：カナ（五十音・濁点・半濁点・長音・読点・句点）と A–Z / 0–9 / 記号
- **音声再生**：WebAudio でビープ音を生成。速度（WPM）と音程（Hz）を調整可能
- **マイク復号**：マイク入力のトーンを検出し、短点/長点・間を分類して復号（速度は自動推定）。適応しきい値による自動ゲインとノイズ耐性、受信レベルメーター付き
- **依存ゼロのコア**：変換ロジックは純粋な TypeScript。UI・音声から独立していてテスト済み

## 仕組み

```
src/
  morse/            # 変換コア（純粋TS・ブラウザ非依存・テスト対象）
    wabun.ts        # 和文モールス表＋濁点/半濁点の分解・合成
    international.ts # 欧文モールス表
    encode.ts       # テキスト → モールス
    decode.ts       # モールス → テキスト
  audio/
    player.ts       # WebAudio でモールスを再生
    detect.ts       # スペクトラム→トーン ON/OFF 判定（純粋・テスト対象）
    classify.ts     # ON/OFF 遷移列→モールス組み立て（純粋・テスト対象）
    decoder.ts      # マイク→上記2つを繋ぐ薄いアダプタ
  app/              # UI
```

### マイク復号のサポート範囲

- 公称 5〜20 WPM の一定速度送信（±20% 程度の揺らぎまで。きれいな信号なら 25 WPM 程度まで動くが保証しない）。
  上限の根拠: 画面更新（60fps）でのサンプリングでは、これ以上速い短点がノイズの瞬きと区別できなくなる
- 対象トーンの帯域内 SNR がおよそ 12dB 以上あること（誤発火防止のゲート）
- 連続トーンの保持は数秒〜十数秒（信号強度に依存。モールスの要素としてはあり得ない長さなので対象外）
- タブが前面にあること（rAF 駆動のため。バックグラウンド受信は将来課題）

受信中の表示は**遡及訂正**される：短点か長点かは両方が観測されて初めて確定するため、
受信初期の符号は後から正しい解釈に置き換わる。速度表示が「測定中」の間は暫定。

### モールスのタイミング

標準比率（単位 = dit）：短点 `1` / 長点 `3` / 符号内の間 `1` / 文字間 `3` / 語間 `7`。
`dit 長 = 1200 / WPM` ミリ秒。

### 和文の濁音・半濁音

和文モールスに濁音そのものの符号は無く、**清音のあとに濁点（`・・`）／半濁点（`・・－－・`）を送る**方式です。
本ツールは送信時に `ガ → カ ＋ ゛` と分解し、受信時に `カ ＋ ゛ → ガ` と合成します。
